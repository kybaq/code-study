<!-- BFS 개념 및 특징

BFS란?
    1) 그래프에서 가장 가까운 노드부터 탐색하는 방식
	2) 큐(Queue) 자료구조를 이용하여 FIFO(First-In-First-Out) 방식으로 구현
	3) 최단 경로, 연결된 요소 탐색, 네트워크 탐색 등에서 많이 활용

BFS의 주요 특징
	1) 가까운 노드부터 탐색하므로 최단 경로 문제에 적합
	2) 모든 노드를 한 번씩만 방문하기 때문에 시간 복잡도 O(V + E)
	3) DFS(깊이 우선 탐색)와 다르게 재귀 호출이 없으며, 큐를 사용하여 탐색 순서를 유지

언제 BFS를 사용해야 할까?
	1) 최단 경로 탐색 (ex. 미로 문제, 최단 거리 구하기)
	2) 모든 노드를 방문해야 하는 경우 (ex. 연결 요소 찾기, 그림 개수 찾기)
	3) 가중치가 없는 그래프에서 최단 거리를 찾을 때 (ex. 다익스트라 알고리즘과 비교)

---


BFS 동작 원리
    1️) 시작 노드를 큐(Queue)에 삽입하고 방문 표시
    2️) 큐에서 노드를 꺼내고, 해당 노드와 연결된 미방문 노드를 큐에 삽입
    3️) 큐가 빌 때까지 반복

---

BFS 탐색 순서 예제

예제 그래프
 1 - 2 - 3
 |   |
 4 - 5

| 큐 상태 | 방문 순서 | 설명 |
|---------|---------|------|
| `[1]` | `1` | 시작 노드 1 삽입 |
| `[2, 4]` | `1 → 2, 4` | 1과 연결된 노드 2, 4 삽입 |
| `[4, 3, 5]` | `1 → 2 → 3, 5` | 2와 연결된 노드 3, 5 삽입 |
| `[3, 5]` | `1 → 2 → 3, 5` | 4 처리 완료 |
| `[5]` | `1 → 2 → 3 → 5` | 3 처리 완료 |
| `[]` | `1 → 2 → 3 → 5 → 끝` | 탐색 종료 |

---


BFS의 시간 복잡도 분석

시간 복잡도: O(V + E)
	1) V: 노드(정점) 개수
	2) E: 간선 개수
	3) 노드마다 1번씩 방문하며, 각 간선을 한 번씩 탐색 → O(V + E)

공간 복잡도: O(V)
	1) 큐에 최대 V개의 노드가 들어갈 수 있음
	2) 방문 배열(visited)도 O(V) 공간 사용

| 비교 항목 | BFS (너비 우선 탐색) | DFS (깊이 우선 탐색) |
|----------|------------------|------------------|
| 탐색 방식 | 가까운 노드부터 탐색 | 깊이 있는 노드부터 탐색 |
| 자료구조 | 큐(Queue) | 스택(Stack) 또는 재귀(Recursion) |
| 최단 경로 | ✅ 최단 거리 탐색에 적합 | ❌ 최단 거리 탐색에 부적합 |
| 사용 사례 | 최단 경로, 그래프 탐색 | 백트래킹, 미로 탐색, 순열/조합 |
